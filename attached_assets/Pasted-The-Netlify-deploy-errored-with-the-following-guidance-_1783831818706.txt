**Diagnosis**

- Relevant log lines: the build runs your npm script [#L57](#L57) which executes the package build script [#L59-L60](#L59-L60), and the build fails because the TypeScript compiler is missing: "sh: 1: tsc: not found" [#L61](#L61).  
- There is also a warning that devDependencies were not installed because NODE_ENV is set (production): [#L15](#L15) and npm warnings about omitting dev deps [#L17-L19](#L17-L19) / [#L58](#L58).  
- Error type & cause: your build script runs `tsc && vite build` but `tsc` is not available in the build environment. Most likely `tsc` (TypeScript) is declared in devDependencies and Netlify did not install devDependencies during the build, so the tsc binary is not present.

Before changing Netlify configuration, verify that TypeScript is present in your repo:
- Check package.json to see whether typescript is listed under devDependencies or dependencies: https://github.com/ANIKETPROJECTS/NIVORAV4/blob/main/package.json

**Solution (pick one)**

Option A — (recommended) Install devDependencies during the Netlify build
1. In your repo’s netlify.toml (or in Netlify UI env vars), set NPM_CONFIG_PRODUCTION to false so npm will install devDependencies and make `tsc` available during build.

Add this to netlify.toml (commit and push):
```toml
[build.environment]
  NPM_CONFIG_PRODUCTION = "false"
```
-or- set an environment variable in the Netlify site UI: NPM_CONFIG_PRODUCTION = false.

This keeps TypeScript as a devDependency (the usual practice) and makes the `tsc` binary available to your build script.

Option B — Move TypeScript into production dependencies
1. Verify TypeScript is declared in package.json (see link above).  
2. If you prefer not to enable installing devDependencies, install TypeScript as a regular dependency so it’s present in production:

Run locally and commit the updated package.json and lockfile:
```bash
npm install typescript --save
git add package.json package-lock.json
git commit -m "Install typescript as dependency for Netlify build"
git push
```

Either approach will ensure `tsc` exists and fix the "tsc: not found" error seen at [#L61](#L61). After applying one of the options, push to the repo and trigger a deploy.

Notes / alternatives:
- If you intended not to run the TypeScript compiler at build time, you can remove `tsc &&` from your build script in package.json so the command becomes just `vite build`. Only do that if you are sure type compilation is unnecessary for your build.
- If you need help changing Node versions on Netlify, see Netlify’s docs: https://docs.netlify.com/configure-builds/manage-dependencies/#node-js-and-javascript
