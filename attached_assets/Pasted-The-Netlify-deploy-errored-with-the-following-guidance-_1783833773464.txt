Error type and cause
- Error: missing Vite CLI when running the build script ("vite: not found") — [line 56](#L56).
- Root cause: the Vite binary was not available in node_modules/.bin because the npm install step failed/was incomplete (see the npm error at [line 17-L20](#L17-L20)). That left the build command unable to execute "vite build". Either Vite is not present in package.json, or it is present but devDependencies were not installed during the Netlify install (or the install crashed).

Solution (step-by-step)

1) Verify Vite is listed in your repo
- Open your package.json in the repo and confirm Vite is present (typically under devDependencies). Example file: https://github.com/ANIKETPROJECTS/NIVORAV4/blob/main/package.json
- If Vite is missing, add it locally then commit:
  - Run:
```bash
npm install --save-dev vite
git add package.json package-lock.json
git commit -m "Add vite"
git push
```

2) Ensure Netlify installs devDependencies during build
- If Vite is in devDependencies (recommended), make sure Netlify installs devDependencies. On Netlify you can set the environment variable NPM_CONFIG_PRODUCTION to false so npm will install devDependencies during the build.
- Add this to netlify.toml (commit the change) or set it in the Build & deploy > Environment > Environment variables in the Netlify UI.

Example netlify.toml snippet:
```toml
[build.environment]
  NPM_CONFIG_PRODUCTION = "false"
```
Or set environment variable in the Netlify site settings UI.

(Alternative: move Vite to "dependencies" with `npm install vite` if you prefer not to modify Netlify environment variables — but keeping it in devDependencies is standard.)

3) Clear Netlify build cache and re-deploy
- Because the logs show an npm error during install ([line 17-L20](#L17-L20)), clear the Netlify build cache and trigger a fresh deploy:
  - In Netlify: Deploys → Trigger deploy → Clear cache and deploy site.

4) Re-run the build and confirm
- After pushing the package.json change or setting NPM_CONFIG_PRODUCTION=false and clearing cache, trigger a new deploy. The build should now find the vite binary and run `vite build` successfully.

Notes
- The npm error "Exit handler never called!" indicates the install may have failed partway; clearing cache and re-running a full install usually resolves that. If you continue to see npm internal errors during install, consider trying a different Node version in Netlify (set via .nvmrc or the Netlify UI). Documentation for changing Node versions on Netlify: https://docs.netlify.com/configure-builds/manage-dependencies/#node-js-and-javascript
