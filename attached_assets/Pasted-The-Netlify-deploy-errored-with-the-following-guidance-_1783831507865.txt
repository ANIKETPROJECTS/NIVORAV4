The Netlify deploy errored, with the following guidance provided:

Diagnosis
- Relevant log lines: the warning about NODE_ENV at [line 15](#L15) and the failing build exit at [line 18](#L18).
- The build is failing because devDependencies were not installed (NODE_ENV is set so npm will skip devDependencies). The project’s build command requires devDependencies (e.g. bundler like Vite/webpack/esbuild). When those tools are missing the build script exits non‑zero.

Cause
- NODE_ENV is set to production (masked in the log) so npm does not install devDependencies during dependency installation. The build command then fails because required build tools from devDependencies are missing.

Solution (choose one)

1) Quick fix — install devDependencies during Netlify build
- In the Netlify UI set an environment variable:
  - Key: NPM_CONFIG_PRODUCTION
  - Value: false
- Or change the Build command to explicitly install devDeps before building:
```bash
npm ci --include=dev && npm run build
```
or
```bash
NPM_CONFIG_PRODUCTION=false npm ci && npm run build
```

2) Persist via netlify.toml
- Add or update netlify.toml in repo root:
```toml
[build]
  command = "npm ci --include=dev && npm run build"

[build.environment]
  NPM_CONFIG_PRODUCTION = "false"
```

3) Verify package.json and build tools
- Verify the build tools are actually listed in package.json (devDependencies) and that package.json is committed:
  - package.json: https://github.com/ANIKETPROJECTS/NIVORAV4/blob/main/package.json
- If build tools are missing, add them (for example, vite, webpack, esbuild) and commit:
```bash
npm install --save-dev vite
git add package.json package-lock.json
git commit -m "Add build tool to devDependencies"
git push
```

Optional: Node version compatibility
- If your project requires a different Node version, ensure Netlify uses the correct Node version (via engines in package.json, .nvmrc, or Netlify env var). See Netlify docs for changing Node versions:
https://docs.netlify.com/configure-builds/manage-dependencies/#node-js-and-javascript

Why this fixes it
- Ensuring devDependencies are installed makes the build tooling available during the build step so the build script can complete rather than exiting with code 2.
