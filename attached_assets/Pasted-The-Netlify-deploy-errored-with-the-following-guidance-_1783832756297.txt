1. Verify typescript is declared in package.json
   - Open your repository package.json and confirm a devDependency entry for typescript exists. (Repo file: https://github.com/ANIKETPROJECTS/NIVORAV4/blob/main/package.json)
   - If typescript is missing, add and commit it:
     ```bash
     npm install --save-dev typescript
     git add package.json package-lock.json
     git commit -m "chore: add typescript as devDependency"
     git push
     ```
   - Then trigger a new Netlify deploy.

2. If typescript is already listed in package.json
   - Ensure package-lock.json is committed to the repo (helps deterministic installs).
   - Update your build command to use npm ci (more reliable for CI) and include dev deps:
     - In netlify.toml change the build.command from:
       npm install --include=dev && npm run build
     - To:
       npm ci --include=dev && npm run build
     Commit netlify.toml and redeploy.

3. Alternative quick fixes
   - If your project does not actually need a pre-build tsc step (for example, if Vite handles TypeScript without a separate tsc compile), update package.json scripts to remove tsc:
     - Edit package.json scripts:
       ```json
       "scripts": {
         "build": "vite build"
       }
       ```
     - Commit and redeploy.
   - Or call tsc via npx so it fails more clearly if missing:
     ```json
     "scripts": {
       "build": "npx tsc && vite build"
     }
     ```

4. If the npm internal error ("Exit handler never called!") persists
   - Try clearing caches and re-deploy:
     - Locally test: npm ci && npm run build
     - On Netlify, try switching to npm ci as noted above.
   - Consider pinning a Node version used by Netlify if your local dev environment uses a different Node version. To force a Node version on Netlify you can add an .nvmrc or set engines in package.json — see Netlify docs for changing Node versions:
     https://docs.netlify.com/configure-builds/manage-dependencies/#node-js-and-javascript
